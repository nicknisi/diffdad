The problem

01Nick runs ~12 coding agents in parallel across repositories and is the review bottleneck. He babysits a dozen tmux panes, forgets to return to finished work, and every change he does review he reads cold as a drive-by. Work sits unreviewed because nothing pulls him back to it, and there is no single place to see what every agent is doing or what is waiting on him.

02Diff Dad today has two disconnected modes — `dad watch` (local working tree, in-flight on this branch) and `dad review` (GitHub PR) — and running them in different places is itself friction. Its core 'PRs into narrated stories' pipeline is too slow for a review loop: PR mode blocks on a full multi-chapter LLM generation (often via the local `claude -p` path, which dad's own loading screen warns is ~minutes and ~5-10x slower than the API) before showing anything, and the resulting prose isn't more revealing than a plain diff. The storytelling concept is valued; its latency and ceremony are not.

03The fix is to make Diff Dad a single, daemon-backed command center that shows all in-flight work and everything awaiting review across repos, renders diffs instantly, and uses a streamlined, streaming guided walkthrough to build genuine understanding of each change — yours and the agents' — rather than narrate at you. This continues and broadens the prior agent-comment-loop ideation (whose watch mode is the foundation it builds on).

Goals ×6
01 One cross-repo home lists every unit — agent-running, your working trees, open PRs, and units awaiting review — so nothing depends on remembering to visit a pane.
02 Opening any unit renders the diff on first paint, with no blocking 'generating' screen; the guided walkthrough streams in over it.
03 The walkthrough is streamlined and streams per section (beat rail + interleaved prose + resolve strips), replacing cold diff-reading with guided understanding.
04 Watch and review stop being separate modes: local working trees, agent units, and PRs are one list distinguished by status, not by command.
05 (Phase 3) The review verdict is produced by a model in a different family than the author, giving an independent second opinion.
06 (Phase 4) Safe units auto-clear without the human (agent opens a PR); only non-clearing units enter the needs-you queue.

01 Review experience (diff-first guided walkthrough) phase · blocking med B+C: beat rail (evolve ChapterTOC.tsx) + streamlined interleaved prose (evolve Chapter.tsx) + inline resolve strips; reuse useScrollTracker for rail tracking. Diff-first non-blocking render (adopt watch mode's pattern in useNarrative for the unit path); per-section streaming. Deliberately switch the default AI path from local claude -p to the API for latency (a user-facing default change). Tests target pure builders/store logic — no DOM test infra exists today, so installing jsdom + @testing-library is a P1 setup step if render-level tests are wanted. Begins unifying watch/review toward one surface.
02 Command center (daemon) phase · blocking high Cross-repo review-unit store (lift the agent-comments/store.ts pattern, key by repo+unit) + submit_for_review MCP tool + a decision-delivery channel back to the agent; needs-you / in-flight / cleared dashboard, status-grouped; tracks agent units, working trees, and PRs as one list. Build the persistent cross-repo list first (generalize server.ts), then launchd terminal-survival as the higher-risk hardening slice. Displays the review pipeline's output, so it follows P1.
03 Cross-model skeptic phase med Independent reviewer on a non-author model family. Primary path: --with codex (local Codex CLI, GPT-class, no API key, reuses existing setup); alternative: a configured OpenAI API provider/model. Note: --with resolves ONLY to local CLIs (claude/codex/pi) per ai-runtime.ts, so it CANNOT select the OpenAI API — reaching that needs a config provider override, not --with. Emits verdict + cited, confidence-scored concerns that replace the same-model triage pass as the source of rail flags + recommended action.
04 Checks + auto-clear gate phase high Run configured/detected build/test/lint/type commands in the worktree; populate the checks row; auto-clear when verdict safe + checks green + 0 to-resolve. Terminal action: the auto-clear/approve decision is delivered to the still-connected agent over P2's decision-delivery channel, and the agent's own tooling opens the PR with the brief as description — dad never touches git. Merge left to human/policy (auto-merge is a stretch opt-in). Per-repo config gates aggressiveness. Depends on P2's decision channel.
05 Notifier + stdio proxy phase low tmux status-line pending count, macOS push (deep-link to a unit), stale escalation, periodic digest. Global one-time MCP registration via a net-new stdio proxy (dad mcp) routing to the daemon — today's MCP is HTTP-only (webStandardStreamableHttp at /mcp), so the stdio transport is from-scratch, not an adaptation. Replaces per-session registration.
